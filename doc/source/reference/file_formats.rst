File Formats
============

MARIO currently works with three main families of user-facing file formats:

Excel workbooks
   Used for full database workbooks, aggregation templates, extensions templates
   and add-sectors workflows.

TXT exports
   Used either as one file per matrix or as canonical flat tables.

Parquet exports
   Used as a binary counterpart to the TXT workflows, including flat exports.

Important distinctions
----------------------

Legacy workbook format
   Public axes often follow ``Region / Level / Item``.

Explicit layout-aware format
   Public axes directly expose the sets carried by a matrix, for example
   ``Region`` plus ``Satellite account`` or ``Region`` plus ``Sector`` plus
   ``Factor of production``.

Workflow-specific templates
---------------------------

* aggregation templates generated by ``get_aggregation_excel(...)``;
* add-sectors workbooks generated by ``get_add_sectors_excel(...)``;
* extension and shock templates generated by the corresponding helper methods.
